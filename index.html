    MY WORK ON GYM PLUS SOFTWARE



    So i had to develop The Objective management and Planning management this is how it works : so at first the client has to make an objective via the form and then select the coach who will be creating the plan for the client then the coach in the dashbord will have a list of objectives that he has to complete so by seing the objective of the client he can create a plan Then he can send the created plan (Exercices plan & Diet plan) to the client by email and also the client can see his plan on his interface. as i was saying this is the concept that i used on my work after that did other feature like :




    Web Application



    .Developed the CRUD for the Objective and the Planning (Symfony,html,css,js)
    .And for the error control on the fields of the forms i used Ajax and the Symfony Assert from Validator
    .Additionally, I implemented FullCalendar JavaScript, allowing clients to view their objectives organized by date, streamlining the management of their objectives.
    .A Chat Bot using Open Ai Api
    .A 3d Gym environment so the Client can see The Gym before paying his fees for the inscription.
    .A Bmi Calculator so the Client can know is he is healthy or not .
    .A Calories Tracker so the Client can have the possibilty to train at home and know how much calories he is loosing so here i used esp32 (Wifi data sending) and ECG sensors and battery (the esp32 will send data 'values of heart rate sound in the blood' to the arduino Application where i will establish a connexion via Wifi so i can get the ip address of the esp32 so i can then use it in my Controller in Symfony to communicate with the esp32 so i can get the data coming ) then in my controller i'm creating a file where i will stock the values coming and then i'm reading the values from the file recnetly created and i'm sending them to chartJs Chart (reading the values from file of 10 seconds passed because the ajax call will lose me 3 to 4 seconds so each 10 seconds i'm having the new values and then send them to a Heart rate equation so i can know the heart rate value because the other equation of Calories Lost is based on it )
    .An Exercice Generator (Client will have several pictures to select from the muscle that he want to train and then he will get a list of exercices with their video and all informations that he need to know. )
    .Additionally i've worked with the chartJs Bundle Offered By Symfony to display The Objectives that a connected coach should do filtered by the Objective Type(Gain Weigh , Lose Weight, Maitain Weight)
    .Also i used the email bundle to send Email to the CLient (email that contains Exercices and Diet Plan)

Then, after 15 seconds, I send the data to be displayed, which is nearly the same approach as in the web project.
